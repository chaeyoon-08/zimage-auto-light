Overview

한눈에 보는 구조

하나의 gcube 워크로드 안에서 여러 노드의 GPU(레플리카)가 이미지를 만들고, 컨테이너 내부 저장소와 마운트되어 있는 클라우드 저장소에 저장됩니다. 워크로드는 서비스 URL로 이 모니터링 페이지를 띄워주고, 페이지는 저장소와 각 GPU 상태를 모아 보여줍니다.

레플리카 = 노드 위 컨테이너
레플리카 한 개는 노드 위에서 도는 컨테이너(GPU 포함)입니다. 한 워크로드에 여러 개가 떠서 같은 작업을 나눠 생성합니다.
두 곳에 저장
각 컨테이너는 자기 내부 저장소와, 워크로드에 마운트된 클라우드 저장소(워크로드 밖)에 함께 저장합니다.
서비스 URL · 이 페이지
워크로드가 서비스 URL로 띄우는 모니터링 페이지. 저장소와 각 GPU 상태를 모아 보여줍니다.
여러 레플리카를 어떻게 구별해서 보여줄까?

서비스 URL 하나 뒤에 GPU가 여러 대인데도 페이지가 각 레플리카를 따로 보여줄 수 있는 건, 각자 자기 ID로 된 파일을 클라우드 저장소의 정해진 위치에 모아 두고, 페이지가 그 위치를 UI 표시용 요청값으로 읽어오기 때문입니다.

  • 각 레플리카(컨테이너)는 고유한 호스트네임(ID)을 가집니다.
  • 각 레플리카는 자기 ID로 된 파일(상태·이미지 메타)을 클라우드 저장소의 정해진 위치(같은 실행 폴더)에 모아 둡니다. 마운트라 저장·읽기가 양방향이에요.
  • 페이지는 그 위치를 UI 표시용 요청값으로 읽어와, ID 기준으로 갈라 어느 레플리카가 무엇을 했는지 구별해 보여줍니다.
  • 생성 요청 자체는 서비스 URL로 가서 그때 응답하는 레플리카가 처리하고, 결과는 위 방식으로 구별돼 표시됩니다.
Tab 01

생성 탭

왼쪽에 입력·작업 제어·레플리카, 오른쪽에 결과 갤러리·자원 패널이 있습니다. 패널을 하나씩 살펴볼게요.

① Manual Generate · 테스트 생성
  • PROMPT·WIDTH·HEIGHT·STEPS·SEED·COUNT를 넣고 생성. SEED를 비우면 무작위.
  • CONFIG 토글 — ON이면 조건 파일 프롬프트로 랜덤 생성(PROMPT 비활성), OFF면 직접 쓴 PROMPT로 생성.
  • 생성 / 결과 — 시작 / 방금 만든 이미지 모아보기. 패널은 접어서 공간을 넓힐 수 있어요.
  • 진행 중인 작업이 있으면 먼저 정리해야 합니다. 아래 Job Status의 취소에서 그 레플리카를 골라 정리한 뒤 생성하세요(자세한 흐름은 ②).
② Job Status · 작업 상태
  • 진행 수(완료/전체)·상태 배지·진행바를 보여줍니다. 기본은 전체 레플리카 합계이고, ③ 목록에서 레플리카를 고르면 그 레플리카만의 진행으로 바뀌어요. 제목 밑 라벨 옆 ✕로 다시 전체로 돌아갑니다.
  • 일시정지·재개·취소 버튼을 누르면 대상 레플리카를 고르는 창이 떠요. 체크박스로 원하는 레플리카(또는 전체 선택)를 골라 적용합니다 — 일시정지는 동작 중(running)인 것, 재개·취소는 일시정지한 것(paused)만 대상이에요. 그래서 처음엔 일시정지 버튼만 활성화되고, 일시정지를 해야 재개·취소 버튼이 활성화됩니다(취소하려면 먼저 일시정지).
  • 진행 중인 작업이 있으면 그 레플리카는 새로 생성되지 않습니다. 그 레플리카를 일시정지한 뒤 취소로 정리하고 생성하세요. (정리 안 하고 생성을 누르면 "일시중지 후 취소하신 다음 다시 진행해주세요" 안내가 떠요.)
  • 파드(레플리카) 자체를 없애는 건 불가합니다 — gcube가 지정 개수를 유지하려 되살리기 때문에, 작업 취소까지가 가능 범위예요.
②-1 타겟 제어 창 (일시정지·재개·취소 공통)
  • 버튼을 누르면 대상이 될 수 있는 레플리카만 목록에 떠요(검색·전체 선택 가능).
  • 원하는 레플리카를 체크하고 확인을 누르면, 그 레플리카(들)에만 명령이 전달됩니다. 서비스 주소는 아무 레플리카로나 연결되지만, 명령은 공유 저장소를 거쳐 고른 대상에게 정확히 전해져요.
  • 저사양 노드에선 명령이 몇 초 뒤 반영될 수 있어요(현재 작업 중인 장이 끝난 뒤 적용).
③ Replicas · 레플리카 목록
  • 돌아가는 레플리카를 검색·선택합니다. All로 전체 보기.
  • 레플리카를 고르면 오른쪽 갤러리·자원과 Job Status 진행이 그 레플리카 기준으로 바뀝니다(상단에 라벨, ✕로 해제). 아무것도 안 고르면 Job Status는 전체 합계예요.
  • 클릭하면 레플리카 상세 모달(대시보드 탭 참고)이 열립니다.
④ Generated Images · 갤러리
  • 만들어진 이미지가 쌓입니다. ALL · AUTO · MANUAL로 거를 수 있어요.
  • 이미지는 화면에 보일 때 불러오고, 한 번 받은 이미지는 캐시되어 다시 볼 때 빠릅니다. 로딩 중엔 표시기가 뜨고, 아직 안 만들어진 이미지는 자동 재시도(계속 안 되면 '다시 불러오기' 버튼)합니다.
  • 썸네일을 누르면 이미지 상세 모달(이미지 탭 참고)이 열립니다.
⑤ Resources · 선택한 레플리카
  • 지금 응답한 GPU의 VRAM·RAM·Util과 이미지 1장 생성 시간(최근·평균·최단·최장)을 보여줍니다.
  • VRAM limit — 한계치(GB)를 넣으면 그보다 많이 쓰는 VRAM이 빨갛게 표시돼, 원하는 사양보다 자원을 더 쓰는 경우를 바로 알아챌 수 있어요.
Tab 02

대시보드 탭

전체 현황을 요약 카드로 보고, GPU별 상태를 카드로 펼쳐 봅니다. 카드를 누르면 그 GPU의 상세 모달이 열립니다.

  • 요약 카드 6개 — 레플리카 수 · 총 생성 이미지 · 상태 분포(실행/멈춤/완료/지연/죽음) · 전체 시간당 생성 · 평균 가동률 · 평균 GPU Util.
  • 레플리카 카드 — GPU별 VRAM·Util·진행률. 이름으로 검색.
  • 검색 조건 — 상태(ALL·RUNNING·DONE·PAUSED), 정렬(이름순·생성 많은순·VRAM 높은순·Util 높은순), VRAM limit 초과만, 죽은 레플리카 포함.
  • VRAM limit — 한계치(GB)를 넣으면 초과한 카드의 VRAM이 빨갛게 표시되고, '초과만' 조건으로 그 카드만 모아 볼 수 있어요.
레플리카 수 색 구분
맨 위 '레플리카' 숫자는 보라 = 살아있음 · 흰색 = 죽음으로 나뉘어 표시됩니다.
보라색 LOADING
모델을 불러오는 중이라는 뜻입니다. 저사양·노하드(디스크리스) 노드에선 더 오래 걸릴 수 있고, 끝나면 자동으로 RUNNING으로 바뀝니다.
화면은 실시간이 아닙니다
네트워크 안정성을 위해 정보를 약간 지연해서 보여줍니다(보통 수 초). 차이는 크지 않지만, 방금 한 조작(생성·일시중지·취소)이 화면에 바로 안 보일 수 있으니 잠깐 기다려 주세요.
레플리카 상세 모달 (카드 클릭)
  • 현재 자원 — VRAM·RAM·Util 게이지와 갱신 상태(초록 정상 / 노랑 지연 / 회색 종료 / 빨강 죽음). 제목 옆에 GPU 정상/이상 배지가 떠서, 갱신은 되는데 GPU만 이상한 경우를 구분해 줘요.
  • 취소·오류 사유 — 작업이 취소·오류로 끝났으면 진행바 아래에 "취소 — N장 후 취소 (N/M)" 같은 사유가 표시됩니다.
  • 작업 시간 — 시작~종료를 타임라인으로(진행 중이면 끝점이 깜빡임).
  • 성능 분석 — 시간당 생성 · 장당 평균 · 가동률 · 이론 최대 · VRAM 효율을 계산 근거와 함께 보여줍니다.
  • 생성 진행 — 진행 수·퍼센트와 평균·최단·최장 생성 시간 통계.
  • 리소스 추이 — VRAM·Util 시계열 그래프(실시간 / 1시간 / 6시간 / 전체).
  • 이 레플리카 이미지 — 미니 갤러리와 더보기(→ 이미지 탭에서 그 레플리카로 필터).
Tab 03

이미지 탭

만들어진 이미지를 한곳에 모아 보고, 검색·세부 조건으로 원하는 것만 골라냅니다. 썸네일을 누르면 상세 모달이 열립니다.

  • 이번 실행 / 전체 — '이번 실행'은 지금 실행이 만든 이미지만, '전체'는 저장소에 쌓인 모든 실행의 이미지를 봅니다.
  • 검색창 — 프롬프트·시드·레플리카·CONFIG 중 무엇이든 한 단어로 빠르게 찾습니다.
  • 검색 조건 — 유형(ALL·AUTO·MANUAL), 정렬(최신·오래된·시드), 그리고 세부 조건(프롬프트·시드·레플리카·CONFIG)을 여러 개 동시에 걸면 모두 만족(AND)하는 것만 보입니다.
이미지 상세 모달 (썸네일 클릭)
  • 생성 파라미터 — 프롬프트·시드·크기·Steps·모델. 프롬프트와 시드는 복사 버튼으로 바로 가져갑니다.
  • 생성 정보 — 유형(AUTO/MANUAL)·CONFIG·실행, 그리고 레플리카(눈 아이콘으로 그 레플리카 상세 모달로 이동)·시작/종료/걸린 시간.
  • 파일 — 파일명(다운로드), 컨테이너·클라우드 경로, 파일 크기.
  • 이 설정으로 다시 생성 — 같은 설정으로 한 장 더 만듭니다(확인 후 진행).

FAQ

자주 묻는 것

생성 버튼이 안 눌려요 (회색이에요)
두 경우예요. ① 생성이 진행 중이면 끝날 때까지 버튼이 잠깁니다('생성 중...'). ② 이전 작업이 남아 있으면 새로 생성되지 않아요. Job Status의 취소에서 그 레플리카를 골라 정리한 뒤 다시 누르세요.
CONFIG를 켰더니 프롬프트가 안 들어가요
정상입니다. CONFIG 토글이 ON이면 조건 파일에서 프롬프트를 뽑아 랜덤 생성하므로, 직접 입력한 PROMPT는 비활성화돼 무시돼요. 내가 쓴 프롬프트로 만들려면 토글을 OFF로 두세요.
Resources에 아무것도 안 떠요
Resources는 레플리카를 선택해야 표시됩니다. ③ 레플리카 목록에서 특정 레플리카를 고르면 Resources(와 Job Status·갤러리)가 그 레플리카로 고정돼 그 레플리카의 자원이 나와요. 고르지 않으면 어느 한 레플리카 값이 혼자 떠서 헷갈리지 않도록 비워두고 안내문만 보여줍니다 — 제목 옆 라벨로 현재 기준을 확인할 수 있습니다.
AUTO 이미지와 MANUAL 이미지는 뭐가 달라요?
AUTO는 워크로드가 자동 생성 설정(조건 파일)으로 백그라운드에서 만든 것, MANUAL은 생성 탭에서 내가 직접 테스트로 만든 것입니다. 갤러리·이미지 탭에서 ALL·AUTO·MANUAL로 걸러 볼 수 있어요.
'이번 실행'과 '전체'는 뭐가 달라요?
이번 실행은 지금 떠 있는 배포(실행)가 만든 이미지만, 전체는 클라우드 저장소에 쌓인 이전 실행까지 모든 이미지를 봅니다. (이미지 탭 상단 토글)
이미지 상세에서 경로가 두 개로 보여요
같은 파일을 두 가지로 보여주는 거예요. 컨테이너 경로(/workspace/outputs/…)는 워크로드 안에서 보이는 전체 경로, 저장소 경로(outputs/…)는 마운트된 /workspace 저장소 안에서의 위치입니다. 마운트는 /workspace 하나라, 저장소에 들어가 outputs 폴더에서 생성된 이미지를 확인할 수 있어요.
레플리카가 '지연' 또는 '죽음'으로 떠요
상태 갱신이 잠깐(기본 120초) 끊기면 먼저 지연(주황)으로 표시돼요 — 아직 살아있고 곧 회복될 수 있다는 뜻이라, 죽음으로 깜빡이지 않아요. 갱신이 오래(기본 5분) 끊겨야 비로소 죽음(빨강)으로 바뀝니다. 죽음은 노드 빠짐·크래시·네트워크 끊김 등 갱신이 완전히 멈춘 경우예요. 사양이 낮은 노드가 잠깐 느린 정도는 지연 단계에서 머물다 회복됩니다. 대시보드의 '죽은 레플리카 포함'으로 확인할 수 있어요.
모르는 레플리카가 '죽음'으로 먼저 떠 있어요
정상입니다. GPU 노드는 붙었다가 반납되고 다른 노드로 교체될 수 있어요. 이때 반납된 옛 노드의 기록이 잠시 죽음(빨강)으로 남아 있을 수 있습니다. 새로 붙은 노드는 모델을 먼저 불러온 뒤(보라 LOADING) 잠깐 있다가 RUNNING으로 뜨기 때문에, 처음엔 죽은 항목만 보이다가 곧 살아있는 레플리카가 나타나요. 죽은 항목이 거슬리면 대시보드의 '죽은 레플리카 포함' 토글로 숨길 수 있습니다.
VRAM limit은 뭘 막아주나요?
막아주는 건 아니고 표시해 줍니다. 원하는 사양(GB)을 넣어두면 그보다 많이 쓰는 GPU를 빨갛게/필터로 알려줘서, 자원을 과하게 쓰는 경우를 빨리 찾을 수 있어요.
이미지가 안 떠요
잠시 후 자동으로 다시 시도하고, 계속 안 되면 '다시 불러오기' 버튼이 나타납니다.
화면이 자동으로 갱신되나요?
네, 수 초마다 자동 갱신됩니다. 이미 떠 있는 내용은 그대로 두고 바뀐 부분만 업데이트됩니다.
